# WHISPER — 晚风集 · 二十四种独处

### Whisper Type Indicator · v0.1

> 1 道前置题 + 约 30 道混合题型（情境题 / 意象题 / 补全题），四大维度交叉 + 一层「显 / 隐」触发 = **32 种状态人格**。每一种都是一张可以被拍成照片的名词。
> 测试的底色不是判断你「是什么样的人」，而是陪你看一眼「最近是什么样的你」。
> 可以分享，可以半年后重测，可以什么都不做只是看一眼就关掉。它不会攻击你，也不会安慰你，它只是在那里。

---

## 〇、这个测试要解决的问题

市面上大部分人格测试在解决同一个问题：**给你一个标签，让你发朋友圈**。它们的语法是断言式的——「你就是这样」「你必须那样」「你最配 XX」。这套语法效率很高，但它有一个副作用：人被简化成了一个符号。

WHISPER 想做一点不一样的事。

它承认三件事：

1. **人是流动的。**同一个人在不同时期会是不同的人，测完半年再测，结果很可能不一样。这不是 bug，这是功能。
2. **有些问题当下就是答不出来的。**不知道、说不清、只是看到了——这些不是「无效作答」，它们本身就是一种答案。
3. **不是所有状态都愿意被说出口。**藏起来的那一部分往往才是真正的底色。测试应该给这一部分留位置，而不是逼人交出来。

WHISPER 的所有设计决策都从这三条出发。如果某个改动和这三条冲突了，那是这三条错了还是改动错了，默认后者。

---

## 一、核心设定：测「最近的你」，不测「永远的你」

和 FWTI / MBTI / SBTI 都不一样，WHISPER 的结果**不是身份标签**，而是**状态切片**。

具体的呈现是「**头固定 · 里流动**」：

- **头（类型名 + 代码）是相对固定的。**一个人半年内重测，大概率会落在同一类或相邻类——这保证了类型系统的识别度和分享性。「我是晚风」这句话可以发朋友圈，可以放在简介里，可以和朋友对代码。
- **里（独白文案 + 小注脚）是每次生成的。**同样测出「晚风」，这次读到的可能是「有点累的晚风」，半年后读到的可能是「刚开始变凉的晚风」。独白基于本次作答的细节（光影分、近远分、跳过次数、漂浮度）微调。

这让 WHISPER 既可以被分享（因为头是稳定的），又每次都有一次性的个人感（因为里是当下生成的）。

## 二、四大维度

WHISPER 不追求心理学严谨——它不是 ECR-R，不是 Big Five，更不是任何一种诊断工具。但四个维度的选择不是随手拍的，每一个都对应了一种**人最基础的内在体验**：

| 维度             | 极性 A                  | 极性 B                  | 对应的内在体验                     | 核心问题                 |
| ---------------- | ----------------------- | ----------------------- | ---------------------------------- | ------------------------ |
| **光影**         | **L（Light · 光）**     | **S（Shadow · 影）**    | 情绪基调：你朝向明的还是朝向暗的？ | 最近你愿意抬头看天吗？   |
| **近远**         | **N（Near · 近）**      | **F（Far · 远）**       | 关系距离：你想靠过去还是走开去？   | 最近你想和人更近还是更远？|
| **动静**         | **M（Motion · 动）**    | **Q（Quiet · 静）**     | 行动欲：你想让事发生还是让事停下？ | 最近你是往前走还是想歇？ |
| **温冷**         | **W（Warm · 温）**      | **C（Cool · 冷）**      | 体感温度：你需要被加热还是习惯体温略低的独处？ | 最近你怕冷吗？        |

四个字母组合 → 2⁴ = **16 种明面类型**。

再叠加一层「**显 / 隐**」——不是通过打分得到，而是通过用户在答题里**选了多少次「说不出」/「暂时答不出来」**决定。跳过少 → 显面（**O**，Open）；跳过多 → 隐面（**H**，Hidden）。**说不出口的那部分，就是你最近真正藏起来的样子。**

最终代码 = 4 字母打分维度 + 1 字母显隐 = **5 字母**，如 `LNMWO`（光近动温 · 显）或 `SFQCH`（影远静冷 · 隐）。

> 16 × 2 = **32 种 WHISPER 类型**。

## 三、三种「非典型」答法 —— 这是本测试的心脏

这是 WHISPER 和 FWTI / SBTI 最本质的区别。传统测试对「非典型答法」的处理非常粗暴：要么强迫选一个，要么合并成一个「跳过」计数。WHISPER 认为**被动至少有三种面貌**，它们的语义完全不同，机制也必须区别对待。

### 3.1 藏 · 「这个问题我暂时答不出来」

用户的心境：**心里其实有答案，但不愿意交出来。**这是隐面的真正来源。

- 不打分
- 跳过计数 **+1**
- 漂浮度不增加
- 跳过 ≥ 3 次 → 触发隐面（类型末位从 O 变成 H）

文案标签统一为：「**这个问题我暂时答不出来**」。不要用「跳过」「不想答」，那两个词太冷。

### 3.2 放 · 「什么也没做，只是看到了」

用户的心境：**看到了，心里有一下，但没采取任何动作。**这不是冷漠（冷漠是没看到），也不是麻木（麻木是看到了但没反应），而是一种**允许事情发生然后过去**的成熟。在情绪心理学里最接近 passive acknowledgment（被动承认）。

- **真实打分**：一般 +影 +静 +冷（允许具体题目微调）
- 跳过计数不增加
- 漂浮度不增加
- 不触发隐面——它恰恰是**没有藏任何东西**的状态

这是整个测试里最接近「禅」的选项，用户常常在答到这类选项时会停一下。要珍惜它，不要滥用——**不是每一题都适合出现「放」**。一般只有「关系温度」「旧事复活」「意外联系」这类题目才适合出这个选项。

### 3.3 晃 · 「都可以，我也不知道」

用户的心境：**心里没有锚点，或者四个选项都像所以都不像。**这是一种暂时失能的状态，在生活里再正常不过，但它不是「藏」。

- 不打分
- 跳过计数不增加
- **漂浮度（Drift）+1**
- 漂浮度 ≥ 5 → 不触发隐面，但在结果页追加一个**小注脚**：

  > *PS：你这次答题的时候，好像有几道题你自己也没想清楚。这很正常——有些日子人就是没有锚点，像一条船在内湖里轻轻晃。今天的你测出来是「XX」，但也许下周再测，你会是别的什么。不用太认真。*

这个注脚本身就是 WHISPER 的产品哲学——**承认测试可能没测准，这种坦诚比结果本身更打动人**。

### 3.4 三种被动的汇总表

| 选项措辞                     | 用户心境         | 打分 | 跳过计数 | 漂浮度 | 隐面触发 |
| ---------------------------- | ---------------- | ---- | -------- | ------ | -------- |
| 这个问题我暂时答不出来       | 藏（有答案不说） | ✗    | +1       | 0      | ≥3 次 ✓  |
| 什么也没做，只是看到了       | 放（看到不动）   | ✓    | 0        | 0      | ✗        |
| 都可以，我也不知道           | 晃（没有锚点）   | ✗    | 0        | +1     | ≥5 次注脚|

用户不需要知道这张表的存在。但他们在答题时会**感觉到被尊重**——因为任何一种当下的心情，都真的有一个选项装得下。这是 WHISPER 的安静承诺。

## 四、打分 · 计分 · 显隐判定

### 4.1 打分

每一道「打分题」的每个选项上都会标记一组维度增量，格式：

```
+L / +S / +N / +F / +M / +Q / +W / +C
```

一个选项可以同时推动多个维度，例如 `+S +Q +C` 表示这个选项同时往「影」「静」「冷」方向加分。

**一般规则**：每次 +1。不使用 FWTI 那种 ±2 的强极化系统——WHISPER 是散文调，不是梗图调，剧烈的 ±2 会让打分感觉像在给用户贴标签。

**特殊规则**：少量「灵魂题」可以让某个维度 +2，作为极强信号。全测试里这种题不超过 5 道。

### 4.2 维度落格

每个维度有一个 counter：

- 光影：`light_score - shadow_score` → 正 = L，负 = S，**0 默认归 S**（下述符号不对称，见 4.4）
- 近远：`near_score - far_score` → 正 = N，负 = F，**0 默认归 F**
- 动静：`motion_score - quiet_score` → 正 = M，负 = Q，**0 默认归 Q**
- 温冷：`warm_score - cool_score` → 正 = W，负 = C，**0 默认归 C**

### 4.3 显 / 隐判定

```
显隐 = H if (跳过次数 >= 3) else O
```

跳过次数 = 用户选择「这个问题我暂时答不出来」的总数。这个数字**不暴露给用户**，也**不编码进分享链接**（理由见 4.5）。

### 4.4 符号不对称 —— 一个必须保留的设计决定

> ⚠️ 和 FWTI 类似，WHISPER 的打分 tie-break **不是对称的**。四个维度平票时默认**全部归到 S / F / Q / C 一侧**，即偏向「影远静冷」。
>
> 这不是 bug。WHISPER 的语言底色就是偏向安静、克制、内省——遇到信息不足时，默认朝这个底色靠。这让整个测试的产品气质比打分结果本身更统一。如果未来有人想改这个规则，请先读完本节再改。

### 4.5 漂浮度

```
drift = 「都可以，我也不知道」的总次数
```

漂浮度 ≥ 5 → 结果页追加小注脚（见 3.3）。漂浮度**不影响**类型判定，只影响文案。

### 4.6 分享链接编码

参考 FWTI 的做法：

- 编码器读取**一个有序的 questionIds 数组**，把每题答案编成一个字符（`0-9` 表选项索引，`-` 表示未答），然后 base64url。
- **跳过计数和漂浮度不编码**——它们是**只属于本次答题的私密信号**，不应该让分享链接的观看者看到「这个人在 3 道题上说不出口」。
- 显 / 隐状态**也不编码**——它从编码中的答案推算出来。这意味着一个人测完 `LNMW` 显面链接分享出去后，别人点开重新解码也会看到显面结果（因为链接不含跳过次数，默认按 0 次计算），但这**不会让原用户的隐面结果丢失**，因为原用户看到的那页是当场生成的。

这条设计的副作用：**链接本身总是显面结果**。这是有意的——隐面是私密的，不给别人看。

> 这个"链接只保留显面"的决策是 WHISPER 和 FWTI 的一个关键分歧。FWTI 的链接是完整的社交货币；WHISPER 的链接是半公开的——能分享的是你愿意被看到的那一面，真正的那一面留在你自己的屏幕上。

## 六、结果页结构（每一种类型的文案模板）

每一种类型的结果页由以下几部分组成，顺序固定：

### 6.1 头图 + 类型名 + 代码

一张竖版极简插画或照片，底下一行字：

```
LFMWO · 晚风
```

字号大但不夸张，字重偏细，不要 bold。

### 6.2 主独白（150 字左右，第二人称）

写作规范见 §7。范例（晚风）：

> 你是一阵晚风。
>
> 不急着去哪里，也不想回哪里。经过有人吃饭的窗户会慢一点，经过还亮着的便利店会再慢一点。你不喜欢被人握住，但你也不真的想离开——只是需要一点点流动的余地，好让那些没说出口的话可以被带走。
>
> 最近你可能有点累，但你不会告诉别人。没关系，风本来就不用解释自己。

### 6.3 三个小板块（每个一句话）

- **此刻最适合做的一件小事**：一句具体的、可以真的去做的建议。不要是「好好爱自己」这种大道理，要是「今晚别再刷手机了，去把阳台的窗开五分钟」这种。
- **最近可能正在回避的一件事**：一句温柔的戳。不要锐评，要像一个很久没见面的朋友轻轻说的那种。
- **如果你是一本书，今天可能翻到了哪一页**：一句纯文学向的收尾。可以是虚构的书名，也可以是真实的书。这一栏是整个结果页的出口，用户读完这一行会合上手机。

### 6.4 条件性追加块

- **漂浮度 ≥ 5**：追加 §3.3 的小注脚
- **跳过次数 ≥ 3（即隐面）**：在主独白之后追加一小段，格式「*你在 X 道题上说不出口。这一面不给别人看也没关系，我知道就好。*」，**不要暴露具体是哪几题**
- **（暂不实现）重测**：如果能读到用户上一次的结果，可以在结果页底部显示一行「*上次你是「月台」，这次你是「晚风」。这几个月，你走过来了一点。*」——这是未来 v0.2 的功能，v0.1 不做

### 6.5 禁止的东西

WHISPER 的结果页**不要**有以下元素，它们都是 FWTI 的语法：

- 最佳 CP / 最差 CP
- 废物指数 / 任何百分数评分
- 各维度柱状图（除非做成极简的水墨风）
- 16 宫格类型表（文艺测试不炫 2⁴ 结构）
- 「认真版解读」这种二元切换。WHISPER 不需要「认真」模式，因为它本来就不是梗图

## 七、风格指南

### 7.1 语言基调

- **第二人称。**用「你」，不用「您」，不用「我」。「你是一阵晚风」>「我是一阵晚风」>「您是一阵晚风」。
- **短句优先，但允许呼吸长短不一。**不要故意押韵，不要对仗工整。工整会变成问卷感，不工整才像人在跟人说话。
- **不用感叹号。**一个都不用。
- **不用网络梗。**不出现「拉满」「绝绝子」「啊这」「DNA 动了」这类词汇。也尽量不出现「你就是那个...」这种断言句式。
- **允许一点点的古意。**偶尔一个「也罢」「便是」之类的字可以，但不要堆。对比 FWTI 的「君」「为君」「之以乃则」——WHISPER 不走那条路，它的文言度 < FWTI。
- **不用 emoji。**结果页一个都没有。题目里也没有。

### 7.2 选项设计原则

这是题目设计的灵魂所在。每一条都要守住。

1. **没有明显正确或更酷的答案。**FWTI 和 SBTI 的选项里经常有一个明显更酷或更可爱的，用户会本能挑那个。WHISPER 必须让所有选项**都不带评价色彩**——每一个都是**一种真实的内心姿势**。如果写完一题发现某个选项明显更招人喜欢，这题就没过关，重写。
2. **选项长度故意不一致。**工整 = 问卷。不工整 = 人话。
3. **具体物件永远大于抽象形容词。**「你是一个怎样的人」太抽象；「你最想放进空房间的是什么」就具体了。参考村上春树、是枝裕和、黎戈的写法。
4. **允许一题出现 5 个选项。**标准题是 4 个选项，但如果某一题需要加「什么也没做，只是看到了」或「都可以，我也不知道」才能真正装下所有内心姿势，就加到 5 个。不要为了工整而牺牲真实。
5. **三种被动选项（§3）的出现位置要有克制。**
   - 「**说不出**」：每一道打分题都可以出现（作为第 5 选项）
   - 「**只是看到了**」：只在「关系题 / 外部事件题 / 意外触发题」里出现，大约 5–8 道
   - 「**都可以**」：只在「意象题 / 选物件题 / 选声音题」里出现，大约 4–6 道

   不要每题都出。这些选项是留白，留白太多就成了空白。

### 7.3 题目三种体裁

WHISPER 的 30 道打分题由三种体裁混排：

**A · 情境题（约 12 题）**：具体生活切片。例：「你在二手书店翻一本旧书，发现扉页上有前一个主人写的一行字。你最希望看到的是？」

**B · 意象题（约 10 题）**：选物件 / 颜色 / 声音 / 天气。例：「下面四种声音，哪一种最像你最近的状态？」

**C · 补全题（约 8 题）**：半句话补全。例：「最近一次我觉得自己还活着，是因为____」

不要让同一种体裁连续出现超过 3 题——要让用户在节奏上感到呼吸变化。

### 7.4 前置题 · 「最近的你，更像哪一种时刻？」

参考 FWTI 的 META 前置题，但语法不同。FWTI 的前置题问「你现在的恋爱状态」，是一个身份标签。WHISPER 的前置题问「你最近像哪一种时刻」，是一个意象。

```
最近的你，更像哪一种时刻？

○ 周日下午三点，阳光很好，你却哪里都不想去
○ 深夜加班完推开公司的门，风很凉
○ 早上醒来，发现手机里没有任何新消息
○ 和熟悉的人吃完饭，走回家的路上
```

这一题**不打分**，但它会**轻微调整后续题目的措辞**（类似 FWTI 的 `variants`），让不同处境的人（工作倦怠 / 独居 / 社恐 / 恋爱中）都能在题目里找到自己。

四种路由分别对应后台的 `routing` 字段：`sunday` / `latenight` / `morning` / `afterdinner`。默认是 `afterdinner`。

## 八、五道样题（v0.1 定稿）

这五道是调性基准，未来写新题时如果感觉跑偏了，回来对照这五道。它们已经在对话中和用户确认过调性。

### 样题 1 · 情境题 · 开场

> 周六早上，你比闹钟早醒了四十分钟。窗帘是拉着的，外面的光透进来一点，是那种还没完全亮起来的灰白色。你最可能做的一件事是——
>
> 1. 继续躺着，什么都不想，就看那点光一点点变 *(+S +Q +C)*
> 2. 起身去厨房烧一壶水，水开之前不看手机 *(+L +Q +W)*
> 3. 翻个身，把窗帘拉得更严，再睡一会儿 *(+S +Q +W)*
> 4. 说不清楚，反正就是醒了，然后时间就过去了 *(漂浮度 +1)*

### 样题 2 · 意象题 · 选声音

> 下面四种声音，哪一种最像你最近的状态？
>
> 1. 雨滴打在空调外机上，有节奏但你数不清次数 *(+S +Q +C)*
> 2. 深夜洗衣机结束时「滴——」的提示音，空房间里格外清楚 *(+S +Q +C，隐面偏向)*
> 3. 楼下早点摊的蒸笼盖被掀开时扑出来的那一下白气 *(+L +M +W)*
> 4. 远远一辆救护车的声音，响了两秒你就听不见了 *(+S +F +C)*
> 5. 都可以，我也不知道 *(漂浮度 +1)*

### 样题 3 · 补全题 · 半句话

> 我最近一次觉得「活着其实也还行」，是因为——
>
> 1. 买到了一样等了很久的东西 *(+L +N +M +W)*
> 2. 在一个没人认识我的地方待了一下午 *(+L +F +Q +C)*
> 3. 有人在我没说话的时候递了一杯水过来 *(+L +N +Q +W)*
> 4. 这个问题我暂时答不出来 *(跳过 +1)*

### 样题 4 · 情境题 · 关系温度

> 一个已经不太联系的朋友，突然在某个平台给你的一条一年前的动态点了赞。你第一反应是——
>
> 1. 有点开心，想回他点什么，又觉得没必要 *(+L +N +Q +W)*
> 2. 点开看了看他的主页，发现他过得还不错，就关掉了 *(+S +F +Q +C)*
> 3. 回他一个表情，就像什么都没发生过 *(+L +N +M +C)*
> 4. 把那条动态删了 *(+S +N +M +C，隐面强触发)*
> 5. 什么也没做，只是看到了 *(+S +Q +C，放)*

### 样题 5 · 意象题 · 选物件

> 如果可以往一个空房间里放一样东西，你最想放的是——
>
> 1. 一盏开着的台灯 *(+L +Q +W)*
> 2. 一扇半开的窗 *(+L +F +Q)*
> 3. 一把没人坐的椅子 *(+S +F +Q +C)*
> 4. 一个装过花、但现在是空的花瓶 *(+S +Q +C，隐面偏向)*
> 5. 都可以，我也不知道 *(漂浮度 +1)*

> 📌 注意样题 5 把「都可以」放在最后而不是最前面。这是有意的——让用户先把前四个选项读完，感受一遍每种物件的气味，再看到「都可以」。这样选「都可以」的人就是**真的四个都读了但还是无法决定**，而不是懒得读。这种人的「不确定」才是真正值得被计入漂浮度的。

## 九、数据结构建议（给未来的实现）

以下是未来写代码时建议的类型定义。不强制，但如果偏离了请在这里更新。

```ts
type Dimension = 'L' | 'S' | 'N' | 'F' | 'M' | 'Q' | 'W' | 'C';
type OptionKind = 'score' | 'skip' | 'see' | 'drift';
type QuestionKind = 'situation' | 'imagery' | 'completion';
type Routing = 'sunday' | 'latenight' | 'morning' | 'afterdinner';

interface Option {
  label: string;
  kind: OptionKind;
  scores?: Partial<Record<Dimension, number>>; // 只在 kind === 'score' 或 'see' 时有
  hiddenBias?: boolean; // 用于「隐面强触发」，仅文档用途
}

interface Question {
  id: number;                         // 稳定 id,append-only
  kind: QuestionKind;
  prompt: string;
  options: Option[];
  variants?: Partial<Record<Routing, {
    prompt?: string;
    options?: Option[];
  }>>;
}

interface Result {
  code: string;            // 如 "LFMWO"
  dimensions: {
    lightShadow: number;   // >0 = L, <0 = S
    nearFar: number;       // >0 = N, <0 = F
    motionQuiet: number;   // >0 = M, <0 = Q
    warmCool: number;      // >0 = W, <0 = C
  };
  skipCount: number;       // 是否进入隐面
  driftCount: number;      // 是否追加漂浮注脚
  routing: Routing;        // 前置题选的 routing
}
```

**编码器注意**：和 FWTI 一样，`questionIds` 数组必须是 append-only。新加题只能追加到末尾，永远不能插入中间，不能重排。否则旧分享链接全部失效。

## 十一、和 FWTI 的关系

WHISPER 和 FWTI 是兄弟项目，不是竞争项目。它们共享一套技术栈（Solid + Vike + 静态部署），可以共享很多底层工具：share-link codec、SSG 路由、主题切换、前置题 variants 机制。但它们的**产品哲学完全相反**：

| 对比项      | FWTI                         | WHISPER                      |
| ----------- | ---------------------------- | ---------------------------- |
| 语法        | 断言式（你就是这样）         | 陪伴式（你最近像这样）       |
| 基调        | 自嘲 · 锐评 · 梗             | 克制 · 留白 · 散文           |
| 人格性质    | 永久标签                     | 当下状态                     |
| 维度        | 4 个（GD/ZR/NL/YF）          | 4 个（L-S/N-F/M-Q/W-C）+ 1 显隐 |
| 类型数      | 16 + 10 隐藏                 | 16 × 2 = 32                  |
| 文言度      | 半文言（君 / 为君 / 之以乃则）| 白话 · 偶尔古意              |
| 分享语法    | 整个结果都可以分享           | 只有显面可分享               |
| 被动选项    | 二元（答 / 不答）            | 三元（藏 / 放 / 晃）         |
| 典型读者反应| 哈哈哈哈 + 截图              | 停顿 · 放下手机              |

两套测试**可以在同一个域名下共存**——用户心情不同时进不同的门。FWTI 治标（想笑），WHISPER 治里（想被看见）。
